## חלק 2 — הבנה לעומק (ארכיטקטורה, קובץ‑קובץ)

### 2.1 שכבות
```
config.js  → supabase.js (window.sb) → api.js (db.*) → store.js (window.store) → מודולים (PAGE_RENDERERS)
                                                     ↘ DEMO: seed בזיכרון
auth.js → הרשאות (roleCaps / canAccess / filterByPermissions)
ui.js (modal/toast/confirm) · picker.js (בורר תלמיד) · app.js (ראוטר + בית)
```

### 2.2 שכבת נתונים
- **`js/config.js`** — `window.CV3`: INSTANCE, שם מוסד, `SUPABASE_URL`, `SUPABASE_ANON_KEY`. `DEMO = !URL || !KEY`. **אין סוד בקוד** מעבר ל‑anon key (RLS מגן).
- **`js/supabase.js`** — יוצר `window.sb` (createClient) או `null` ב‑DEMO.
- **`js/api.js`** — `window.db` = `list/insert/update/remove(table, …)` מעל Supabase; ב‑DEMO מחזיר ריק (ה‑store הוא שמחזיק את ה‑seed).
- **`js/store.js`** — **`window.store`** = `list/add/update/remove/byStudent`. ב‑DEMO: seed עשיר בזיכרון (משוכפל, עם רצפי id). חי: מנתב ל‑`window.db`. **כל המודולים עובדים מול `window.store` בלבד** — זו נקודת המעבר בין DEMO לחי.

### 2.3 אימות והרשאות — `js/auth.js`
- `doLogin()` — DEMO: התאמה מול `users` (שם/ת״ז/טלפון + סיסמה). חי: `sb.auth.signInWithPassword` (מייל סינתטי `<שם>@bht.co.il`).
- `roleCaps(role)` — מגדיר `perms` (מסכים מותרים), `mode` (full/readonly/writeonly), `scoped` (מחנך = כיתתו).
- `setUser` — טוען scope כיתות (`user_class_access`), מוסיף `body.class` (mode‑readonly/writeonly ל‑CSS), קורא ל‑`filterByPermissions` (מסתיר אריחים אסורים) ומציג בית.
- `canAccess(id)` — שער גישה למסך (נבדק ב-`app.js showPage`).
- **החשיפה האמיתית של הנתונים היא ה‑RLS בצד‑שרת** (`supabase/policies.sql`); הצד‑לקוח הוא UX בלבד.

### 2.4 ראוטר ובית — `js/app.js`
- `MODULES` — הגדרת 11 האריחים (id/label/icon/group/color/adminOnly).
- `buildTiles` / `buildPages` — בונה רשת אריחים ועמודי‑שלד.
- `showPage(id)` — בדיקת `canAccess` → הצגת העמוד → הרצת `PAGE_RENDERERS[id]`.
- `updateHomeMode()` — מורה→בית‑מורה, אחר→רשת + **`renderHomeReports()`** (הפיד + באנר; נוסף בסבב זה).

### 2.5 מודולים (כל אחד רושם `window.PAGE_RENDERERS[id]`)
